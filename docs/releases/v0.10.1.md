# skillwiki v0.10.1

Patch release for live managed-write blockers found after v0.10.0 fleet redeploy.

## Fixes

- **Pull helper resolution:** packaged npm installs resolve `wiki-pull-with-auto-resolve.sh` from `dist/vault-sync/scripts/` (same directory as `dist/cli.js`). Host-install fallbacks: macOS `~/Library/Application Support/vault-sync/bin/`, Linux `~/.local/share/vault-sync/bin/`.
- **Stale review-required journals:** managed-write preflight auto-supersedes handoff journals when the worktree is clean, Git is idle, and `target_oid` is an ancestor of HEAD.
- **Operator CLI:** `skillwiki sync journal list` and `skillwiki sync journal clear-stale [--dry-run]`.
- **Doctor:** checks `vault_sync_pull_helper` and `vault_sync_review_required_journals`.
- **Update:** prints Migration 0.10.1 notes when upgrading from &lt;0.10.1.

## Migration (0.10.0 → 0.10.1)

```bash
npm install -g skillwiki@0.10.1
# or: skillwiki update

skillwiki doctor

# if still blocked by journals and vault is clean:
skillwiki sync journal list
skillwiki sync journal clear-stale --dry-run
skillwiki sync journal clear-stale

# legacy only if helper still missing:
# export SKILLWIKI_VAULT_SYNC_PULL_HELPER=.../wiki-pull-with-auto-resolve.sh
```

Redeploy vault-sync on fleet hosts (macos-dev, sg02, sg01) with `VS_PACKAGE_VERSION=0.10.1`.
