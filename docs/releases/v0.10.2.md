# skillwiki v0.10.2

Patch release that closes the remaining verification gaps for managed root
projections introduced in v0.10.0.

## Fixes

- **Deterministic snapshot tests:** live snapshot fixtures now create a valid
  vault before projection materialization, and guard-specific tests use their
  own `skillwiki` wrapper instead of depending on a global executable.
- **Linux shell gate in CI:** GitHub Actions now builds the CLI and runs the
  complete `npm run test:vault-sync` suite on Ubuntu.
- **One root-index universe:** root-index rendering, `index-check`, and the
  `index_incomplete` lint bucket now use the same schema-valid typed pages and
  immediate project README entries.
- **Clear rejected-page diagnostics:** invalid typed pages remain visible in
  their frontmatter or path lint buckets instead of being reported as missing
  index entries.
- **Path-aware compatibility:** compound pages remain known compatibility
  targets, and duplicate basenames keep their full-path identity.

## Rollout and verification

After upgrading the CLI and vault-sync runtime, verify the exact release
identity before any live projection write:

```bash
npm install -g skillwiki@0.10.2
skillwiki --version
skillwiki doctor

VS_READ_ONLY=1 VS_JSON=1 \
  bash packages/vault-sync/skills/vault-sync-status/status.sh --read-only --json
```

The installed runtime manifest must report package version `0.10.2`, the
release commit, and matching package-owned hashes. Only the configured
projection authority may write root `index.md` and root `log.md`; all other
hosts remain preview-only for those files.
